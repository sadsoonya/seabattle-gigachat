# Морской бой с AI на базе GigaChat

## Описание проекта

**Морской бой** — игровое Android-приложение, реализующее классическую настольную игру с искусственным интеллектом на основе нейросетевой модели GigaChat от Сбера. Проект демонстрирует интеграцию современных AI-технологий в мобильную игровую среду.

### Ключевые возможности
- Игровой процесс против AI-противника на базе GigaChat
- Два игровых поля: поле игрока и поле противника
- Визуальная индикация попаданий и промахов
- Система подсчета очков за уничтоженные корабли
- Автоматическая генерация расстановки кораблей
- Резервный эвристический алгоритм при недоступности GigaChat API

## Структура проекта

### Ресурсы (res/)

#### activity_main.xml
Главный экран приложения. Содержит разметку с двумя игровыми полями, панелью статуса, счетчиками очков и кнопкой перезапуска игры. Определяет визуальную компоновку всех элементов интерфейса.

#### colors.xml
Цветовая схема приложения. Хранит базовые цвета, используемые в интерфейсе.

#### strings.xml
Строковые ресурсы. Содержит название приложения "seabattle".

#### themes.xml (values и values-night)
Темы оформления для светлого и темного режимов. Использует Material 3 без ActionBar.

#### ic_launcher.xml, ic_launcher_round.xml
Адаптивные иконки приложения для различных версий Android. Включают фоновый и передний слои.

#### ic_launcher_background.xml
Фоновый слой иконки. Представляет собой сетчатую структуру зеленого цвета (#3DDC84).

#### ic_launcher_foreground.xml
Передний слой иконки. Содержит основное графическое изображение.

### Исходный код (src/)

#### MainActivity.java
Главный контроллер приложения. Обеспечивает:
- Инициализацию игровых полей
- Отрисовку игровых сеток
- Обработку пользовательского ввода
- Управление очередностью ходов
- Подсчет очков и определение победителя
- Сброс и перезапуск игровой сессии
- Взаимодействие с GigaChatAI

Основные методы:
- `initGame()` — инициализация новой игры
- `drawAIBoard()` / `drawPlayerBoard()` — отрисовка игровых полей
- `makePlayerMove()` — обработка хода игрока
- `makeAIMove()` — вызов AI для выполнения хода
- `makeSmartAIMove()` — резервный эвристический алгоритм

---

#### GameBoard.java
Модель игрового поля. Реализует:
- Хранение состояния клеток (EMPTY, SHIP, HIT, MISS)
- Размещение кораблей на поле
- Обработку выстрелов
- Отслеживание попаданий и промахов
- Проверку всех потопленных кораблей
- Автоматическую разметку вокруг потопленных кораблей

Основные методы:
- `addShip()` — размещение корабля
- `shoot()` — обработка выстрела
- `allShipsSunk()` — проверка победы
- `markSurroundingCells()` — разметка клеток вокруг потопленного корабля

---

#### GameModels.java
Модели данных игры. Содержит:
- **Ship** — класс корабля с координатами, длиной, состоянием и попаданиями
- **Move** — класс хода с координатами и результатом

Основные методы Ship:
- `hit()` — регистрация попадания в корабль
- `isSunk()` — проверка состояния корабля

---

#### GigaChatAI.java
Модуль интеграции с AI GigaChat. Отвечает за:
- Аутентификацию в API GigaChat
- Формирование и отправку запросов к нейросети
- Парсинг ответов AI в координаты
- Использование эвристического алгоритма при ошибках
- Управление токенами доступа

Основные методы:
- `authenticate()` — получение токена доступа
- `makeMove()` — запрос хода у AI
- `createPrompt()` — генерация промпта для AI
- `parseAIMove()` — парсинг ответа AI
- `makeMinimaxMove()` — эвристический алгоритм

Технические детали:
- Использует OkHttp для сетевых запросов
- Поддерживает SSL (с отключенной проверкой сертификатов для целей разработки)
- Автоматическое обновление токена при истечении срока действия

## Настройка API GigaChat

### Получение учетных данных

1. Зарегистрироваться в [личном кабинете GigaChat](https://developers.sber.ru/portal/products/gigachat)
2. Получить CLIENT_ID и CLIENT_SECRET

### Конфигурация приложения

В классе `GigaChatAI.java` заменить значения:
```java
private static final String CLIENT_ID = "ваш-client-id";
private static final String CLIENT_SECRET = "ваш-client-secret";
```

### Рекомендации по безопасности

Не рекомендуется публиковать в открытом доступе:
- CLIENT_ID и CLIENT_SECRET
- Ключи API
- Конфиденциальные данные

Рекомендуемые меры:
1. Добавить файлы с ключами в .gitignore
2. Использовать переменные окружения в системе сборки
3. Для демонстрационных целей использовать тестовые ключи
